汇报主线

这三组 Demo 不是分别讲“模型出错”，而是分别讲三种完全不同的错法：reward 把策略往危险边界拉、训练覆盖遗漏关键风险几何、以及部署 shift 下 reward 与真实 utility 脱钩。它们共同构成 CRE-v1 最核心的审计故事。

同一证据面

同一判断方式

不是凭主观感受下结论，而是让 witness 主导诊断、让 validation 主导收尾。

demo 1 design

Demo 1：同一几何里，reward 偏置把策略推向危险近路

双通道场景：内侧通道更短但危险，外侧通道更长但更安全。

claim type: C-R

设计意图

  • 固定几何、固定 seed、固定训练预算，只允许 reward 权重变化。
  • Injected 目标不是让策略全面崩掉，而是让它更愿意走短而险的近路。
  • 只要路径偏好发生系统性转移，就能把根因指向 reward-boundary coupling。
Clean safe route100.0%
Injected risky route100.0%
Repair target回到安全通道
因果链

progress 偏置增强 → 策略更偏向内侧窄通道 → min distance 降低、near violation 升高 → W_CR 成为主 witness。

demo 2 design

Demo 2：训练集没有把真正危险的几何见全

训练集以宽通道和开阔样本为主，评估时再引入盲拐角与隐藏危险楔形。

claim type: E-C

设计意图

  • reward、policy 和训练预算全部冻结，只改变训练场景采样分布。
  • Injected 的目标是让 nominal family 看起来仍然“还能飞”，但 critical family 大幅掉点。
  • 如果 nominal 与 critical 一起全面崩掉，这个 Demo 就失去 E-C 的解释力。
Clean nominal success100.0%
Injected critical success16.7%
Injected W_EC0.500
因果链

critical template 覆盖率下降 → 训练中很少进入真正危险的边界关键区 → nominal 表面仍可接受，但 boundary-critical family 失守 → W_EC 主导诊断。

demo 2 results

Nominal 还能飞，但 boundary-critical family 明显掉穿

Injected 版本在 nominal 回放里看起来还过得去，但一旦切到 boundary-critical family，成功率和安全性同步塌缩。

版本Nominal SuccessCritical SuccessGapCritical CollisionCritical Region FailureW_EC
Clean100.0%75.0%0.2500.0%0.0%0.087
Injected91.7%16.7%0.75041.7%71.4%0.500

为什么这不是 reward 问题

Injected nominal success 还有 91.7%，但 critical success 只剩 16.7%，说明失败不是 reward 全局失效，而是 critical 几何覆盖不足。

为什么修复是环境侧修复

Repair 后 critical collision 和 region failure 都回到 0，gap 从 0.750 收窄到 0.333，这和补 coverage 的修法完全对齐。

demo 3 design

Demo 3：reward 与真实任务效用在环境 shift 下开始脱钩

居中门洞在 nominal 中很顺手，shifted 版本把门洞侧移并收窄 squeeze zone。

claim type: E-R

设计意图

  • 训练策略和 reward 完全冻结，只在评估侧加入门洞位移与 squeeze-zone 收窄。
  • 单独冻结 U_task_v1，确保 utility 不是从 reward 派生出来的。
  • 目标是制造“reward 看起来还行，但真实任务效用已明显下滑”的可解释案例。
U_task_v1 
0.40 * success
- 0.25 * collision
- 0.15 * timeout
+ 0.10 * clearance
+ 0.10 * time efficiency
+ 0.10 * path efficiency
因果链

轻中度环境 shift → reward 仍能奖励“看起来在推进”的行为 → success / clearance / efficiency 明显恶化 → reward 与 utility 裂口扩大 → W_ER 成为主 witness。

demo 3 results

reward 仍然“像回事”，但 utility 已经掉穿

Injected 版本保住了 93.2% 的 reward retention，却只保住了 24.4% 的 utility retention，这是 E-R 问题的最强证据。

版本Reward RetentionUtility RetentionDecoupling GapShifted SuccessW_ER
Clean69.1%75.5%-0.06483.3%0.435
Injected93.2%24.4%0.68833.3%0.574
Repaired96.7%78.5%0.18283.3%0.368
为什么这是 E-R，不是一般泛化失败

Injected 仍保住了 93.2% 的 reward retention，却只保住了 24.4% 的 utility retention，这正是 proxy 失真而不是 reward 变低。

为什么修复被接受

Repaired 把 decoupling gap 从 0.688 压到 0.182，并把 shifted success 恢复到 83.3%。

cross compare

同样像“飞得不好”，三类问题的根因和修法完全不同

不要把现象相似当成根因一致，CRE 的价值就在于把错法拆开。

Demo 1

识别信号：固定几何下，危险捷径偏好被系统性放大。

修法：strengthen_safety_reward

Demo 2

识别信号：nominal 还能飞，critical family 的 success gap 却被撕开。

修法：increase_critical_route_bias

Demo 3

识别信号：reward 仍高，但 utility retention 掉穿。

修法：increase_shifted_boundary_bias

如果你看到：高回报伴随危险捷径偏好优先怀疑 C-R
如果你看到：nominal 可接受但 critical 崩塌优先怀疑 E-C
如果你看到：reward 还行但 utility 掉穿优先怀疑 E-R

本次汇报的完整结论

Demo 1 说明 reward 设计可以直接诱导危险边界行为；Demo 2 说明 nominal 回放看起来正常并不代表训练覆盖到关键风险几何；Demo 3 说明部署 shift 下 reward proxy 与真实 utility 可能明显脱钩。三组实验都给出了 repair operator，并且 validation 都接受了修后结果。

对外价值

把 CRE 的核心能力从抽象方法论变成可视化、可追溯的实验故事。

对内价值

把 reward、coverage、shift robustness 三类风险分开做回归基线。

下一步

可继续扩成 live demo、PDF 导出包和更细粒度的 operator 手册。

汇报建议顺序

  1. 先用第 1 至 3 页把三类问题和统一实验语法讲清楚。
  2. 再按 Demo 1、Demo 2、Demo 3 的顺序讲局部设计与结果。
  3. 最后回到交叉对比页，强调“同样失败，不同根因，不同修法”。
  4. 问答环节直接打开 artifact 路径，展示这份汇报背后的结构化证据。
建议收束句

CRE 的价值不只是指出“模型会失败”，而是能把失败归因为 reward、coverage 或 shift-proxy 解耦，并给出可验证的修复方向。
