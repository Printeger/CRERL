Stage ④ — Diagnostic Aggregation

Diagnostic Products

ranked inconsistency types · witness evidence · flags · uncertainty intervals

Stage ⑤ — Repair Proposal

Reward-space

Stage ⑥ — Acceptance & Validation

Acceptance Criteria

① Ψ_CRE(S′) > Ψ_CRE(S) ② Safety(S′) ≥ Safety(S) ③ U_nom(S′) ≥ U_nom(S) − ε_perf ④ d_spec(S, S′) ≤ η_edit

CONSISTENT

Ψ_CRE ≥ τ_alarm
no repair required

ACCEPT

validated repaired spec S′ adopted

HARD_REJECT

repair loop exceeds N_max
human alert with best-seen proposal

PRECHECK_FAIL

fatal precheck violation before estimation

PO

Pipeline Orchestrator

Validate CFG → sequence all modules → manage audit trail → emit PipelineResult

M1

Specification Parsing

parse NL input → detect ambiguities → run symbolic precheck

PRECHECK_FAIL

M1 precheck halted pipeline

Continue

typed spec survives ambiguity escalation and precheck

M2

Primary Estimators

compute canonical reporters and supplementary diagnostics

M3

Enhanced Estimators

compute κ̃_CR, γ̃_EC, δ̃_ER; route only to discrepancy protocol

DP(M6)

Discrepancy Protocol

Case A → LATENT_INCONSISTENCY + widened CI + M6 temporal hypothesis; Case B → CRITIC_QUALITY_WARN; else pass-through

M5

Composite Scoring

compute Ψ_CRE and bootstrap CI after M2 / M3 / DP

CONSISTENT

if Ψ_CRE ≥ τ_alarm
exit without repair

Else continue

if Ψ_CRE < τ_alarm, enter semantic analysis and repair loop

M4

Semantic Analysis

generate failure hypothesis and repair targets; must not modify numerical fields

M7

Repair Generation

generate proposals → minimality filter → rollout-first ranking / llm tiebreaker

rejection_feedback ↺
M8

Acceptance Loop

evaluate C1–C4 + semantic consistency; accept or regenerate next iteration

ACCEPT

validated SpecS′ adopted

HARD_REJECT

loop exceeds N_max
emit human alert

SpecS

immutable structured specification produced after M1; versioned on accepted repair

DiagReport

canonical + supplementary + enhanced reporters, ψ score, CI, flags, discrepancy, failure hypothesis

RepairProposal

spec_prime, operator_class, declared_side_effects, semantic justification, predicted / rough delta ψ

AuditTrail

append-only per-module record stream with input/output hashes and iteration metadata

AcceptanceVerdict

accepted, c1–c4 flags, semantic score, intent preservation, rejection feedback
