Current Package Shape

Release-grade demo path already exists

The repository now contains a minimal benchmark suite with one clean case and one injected case for each inconsistency type, plus an integration-aware release bundle.

Class I

C-R

Reward-constraint conflict on the nominal scene family. Best explained through reward terms, margin-seeking behavior, and reduced obstacle clearance.

Class II

E-C

Coverage mismatch on boundary-critical scenes. Best explained through failure on hard edge cases that were underrepresented during normal operation.

Class III

E-R

Shift fragility between nominal and shifted families. Best explained through a transfer gap in success rate and obstacle clearance.

1

Open with the scene

Use test_flight.py in GUI mode to show the UAV, start-goal markers, and the scene family. This anchors the audience in indoor obstacle avoidance before any metrics appear.

2

Move to the metrics

Use the local dashboard to display success rate, collision rate, min distance, near-violation ratio, and the relevant witness: W_CR, W_EC, or W_ER.

3

Close with the artifact bundle

Show the report, repair, and validation outputs to prove that the pipeline did not stop at detection. The final release bundle provides the clean-vs-injected demo contract.
